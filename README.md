## 이슈 업로드

### 우리가 만드는 프로젝트와 관련된 모든 이슈는 [본인이 만든 레포지토리의 issue 탭]에 업로드한다.

    
**1. issues 탭에서 new issue를 누른다.**

<img width="1281" alt="스크린샷 2022-10-21 오후 4 29 13" src="https://user-images.githubusercontent.com/17779284/197138458-ba9c796d-2b3d-4d81-b047-a6372f636d30.png">

**2. 템플릿을 선택한다.**

<img width="1283" alt="스크린샷 2022-10-21 오후 4 29 57" src="https://user-images.githubusercontent.com/17779284/197138623-8560c63f-ad4a-473c-8d4e-bb233af8926d.png">


**3. 제목은 [기기명-브라우저명-가로사이즈-페이지명] 이슈 안건 형태로 작성**

예) [mac-chrome-414px-메인] 게시글 아이템의 태그에서 #이 두개가 보여지는 현상


**4. 내용에는 1. 이슈 내용, 2. 이미지 및 영상, 3. (이슈 발생한) 링크 첨부등이 있다.**

필수 첨부 : 링크 
선택 첨부 :이미지 및 영상 (최대한 권장)

템플릿에 있는 내용을 작성한다.

**5. 게시글 업로드전에 라벨링을 해야한다.**

<img width="309" alt="스크린샷 2022-10-21 오후 4 31 05" src="https://user-images.githubusercontent.com/17779284/197138857-edcc0df6-ef86-460a-9587-72e4d51c3018.png">

Assignees : 이슈를 올리는 사람 판단하에 이슈를 해결해야할 것 같은 사람을 의미한다
    
Labels : 이슈를 올리는 사람 판단하에 이슈라벨을 붙임
    
’버그-심각’과 ‘버그-일반’
    
’버그-심각’은 페이지나 기능이 아예 동작하지 않을 때 사용
    
’버그-일반’은 동작은 하는데 기획과 달리 동작을 하거나 애매할 때 사용
    
<img width="667" alt="스크린샷 2022-10-21 오후 4 31 29" src="https://user-images.githubusercontent.com/17779284/197138918-6eb0fce6-0418-43e5-879f-1783b4010033.png">

Milestone : 목표 기준을 정해준다. (mvp 개발시, v2버전개발시 등 그 때 상황에 따라 변경됨)

**6. submit new issue 버튼을 클릭하여 이슈를 게시한다**
  
            
#### 7. git commit 시 이슈 아이디로 close를 한다. 예) git commit -m "[closed #1] ㅇㅇ한 이슈 해결"
